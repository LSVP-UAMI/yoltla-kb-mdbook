[#descripcion]
== DescripciГіn

El benchmark High Performance Conjugate Gradients (HPCG) es un esfuerzo por crear una nueva mГ©trica para clasificar los sistemas HPC. HPCG pretende ser un complemento del benchmark High Performance LINPACK (HPL), que actualmente se utiliza para clasificar los sistemas informГЎticos https://www.top500.org/[TOP500]. 

Los patrones computacionales y de acceso a datos de HPL siguen siendo representativos de algunas aplicaciones escalables importantes, pero no de todas. HPCG estГЎ diseГ±ado para ejercitar patrones computacionales y de acceso a datos que coincidan mГЎs estrechamente con un conjunto diferente y amplio de aplicaciones importantes, y para incentivar a los diseГ±adores de sistemas informГЎticos a invertir en capacidades que tendrГЎn un impacto en el rendimiento colectivo de estas aplicaciones.

Para obtener mГЎs informaciГіn, visite el sitio oficial de https://hpcg.info/[HPCG].


[#archivo_entrada]
== Archivo de entrada
HPCG necesita un archivo de entrada para poder ejecutarse, por defecto HPCG buscarГЎ este archivo con el nombre _hpcg.dat_. A continuaciГіn se presenta un ejemplo de este archivo:

.hpcg.dat
----
HPCG benchmark input file
Sandia National Laboratories; University of Tennessee, Knoxville
104 104 104
60
----

Este archivo consta de 4 lГ­neas y contiene informaciГіn sobre el tamaГ±o del problema, la configuraciГіn del sistema y las caracterГ­sticas del algoritmo que utilizarГЎ el ejecutable. A continuaciГіn se explica cada una de las lГ­neas que conforman este archivo:

* *LГ­nea 1:* Sin usar. Puede utilizar esta lГ­nea para el fin que desee. Por ejemplo, podrГ­a usarla para resumir el contenido del archivo de entrada. El valor por defecto de esta lГ­nea es:
+
----
HPCG benchmark input file
----

* *LГ­nea 2:* Sin usar. Igual que la lГ­nea 1. El valor por defecto de esta lГ­nea es:
+
----
Sandia National Laboratories; University of Tennessee, Knoxville
----

* *LГ­nea 3:* Esta lГ­nea especifica las dimensiones locales (a un proceso MPI) del problema. El valor por defecto de esta lГ­nea es:
+
----
104 104 104
----
+
lo que significa que cada proceso MPI calcularГЎ una soluciГіn para un cubo de tamaГ±o 104 puntos.
+
[WARNING, caption=IMPORTANTE]
====
El tamaГ±o del problema debe ser un mГєltiplo de 8.
====

* *LГ­nea 4:* Esta lГ­nea especifica la cantidad de segundos que debe ejecutarse la parte cronometrada del benchmark. El valor por defecto de esta lГ­nea es:
+
----
60
----
+
lo que significa que la parte cronometrada del benchmark durarГЎ 1 minuto. Este perГ­odo de tiempo no es suficiente para enviar una ejecuciГіn oficial, pero brinda datos suficientes para ajustar el punto de referencia en la mayorГ­a de los casos.
+
[NOTE, caption=NOTA]
====
Las ejecuciones oficiales deben ser de al menos 1800 segundos (30 minutos).
====

Para lanzar pruebas de forma individual, ubГ­quese dentro del directorio de la prueba de interГ©s, y ejecute el comando:
