[#descripcion]
== DescripciГіn

La velocidad de escritura de un disco es el parГЎmetro que determina la rapidez con la que se puede escribir un archivo en el sistema. Una baja velocidad de escritura puede ser signo de un disco defectuoso o de una configuraciГіn inadecuada, por lo que es importante conocer el valor de este parГЎmetro.

Existen diferentes programas y utilidades para determinar este valor, sin embargo, muchos de ellos requieren permisos de superusuario (root) para su ejecuciГіn, con el fin de evitar este problema, se optГі por utilizar el comando `dd` del del paquete https://www.gnu.org/software/coreutils/[GNU Core Utilities].


[#ejecucion]
== EjecuciГіn

1. Cargue el mГіdulo de Coreutils:
+
[source, shell]
----
[t.800@yoltla ~]$ module load coreutils-8.32-gcc-7.2.0-43jmyud
----

2. Ejecute el comando:
+
[source, shell]
----
[t.800@yoltla ~]$ sync; dd if=/dev/zero of=/tmp/write_file bs=1M count=1024 oflag=dsync status=progress; sync
----

[#lanzar_pruebas]
=== Lanzar pruebas

[#individualmente]
==== *Individualmente*

Para lanzar pruebas de forma individual, ubГӯquese dentro del directorio de la prueba de interГ©s, y ejecute el comando:

[source, shell]
----
reframe -c <nombre_script> -r
----

Por ejemplo, para lanzar la prueba de los nodos NC, ejecute el comando:

[source, shell]
----
[t.800@yoltla nc]$ reframe -c diskwrite_nc.py -r
----


[#etiquetas]
==== *Etiquetas*

Utilizando etiquetas puede lanzar mГәltiples pruebas con un solo comando. Para lanzar todas las pruebas, siga los siguientes pasos:

. UbГӯquese en el directorio raГӯz _diskwrite_:
+
[source, shell]
----
[t.800@yoltla diskwrite]$ 
----

. Cree el directorio _logs_:
+
[source, shell]
----
[t.800@yoltla diskwrite]$ mkdir logs
----

. Ejecute el comando:
+
[source, shell]
----
[t.800@yoltla diskwrite]$ reframe -c . -R -t disk -t write -r
----
